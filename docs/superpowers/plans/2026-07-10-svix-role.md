# Svix Role — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extrair Redis + Svix Server para uma role Ansible `svix` dedicada, com secrets no inventário/vault, e deixar DEV05 apenas com Korp.Webhook dependendo dessa role.

**Architecture:** Role standalone no padrão Temporal: compose próprio (`svix-compose.yml`) iniciado pela role; secrets `svix.jwt_secret` / `svix.main_secret` gerados no `inventory-playbook.yml`; DEV05 declara `dependencies: [svix]` e referencia os secrets no Consul KV. Sem publish de porta; imagem pinada em `v1.84`.

**Tech Stack:** Ansible, Docker Compose v2, `svix/svix-server:v1.84`, Redis 7 alpine, `community.docker.docker_compose_v2`, `utils/create_db/postgres`

**Spec:** `docs/superpowers/specs/2026-07-10-svix-role-design.md`

**Escopo:** apenas `KorpSetupLinux` (não alterar `infrastructure-setup-dev`)

## Task 2: Compose `svix-compose.yml.j2`

**Files:**
- Create: `roles/svix/templates/composes/svix-compose.yml.j2`

- [ ] **Step 1: Criar o template do compose**

```yaml
version: "3.8"

networks:
  servicos:
    external:
      name: servicos

services:
  redis_svix:
    image: "{{ svix_redis_image }}"
    container_name: "redis_svix"
    command: "--save 60 500 --appendonly yes --appendfsync everysec"
    volumes:
      - "{{ dados_docker_dir_path }}/svix_redis/:/data"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 1s
      timeout: 1s
      retries: 600
    networks:
      - servicos

  svix:
    image: "{{ svix_image }}"
    container_name: "svix"
    depends_on:
      redis_svix:
        condition: service_healthy
    environment:
      SVIX_DB_DSN: "postgres://{{ postgres.korp_user }}:{{ postgres.korp_password }}@{{ postgres.address }}:5432/{{ svix_db_name }}"
      SVIX_REDIS_DSN: "redis://redis_svix:6379"
      SVIX_WHITELIST_SUBNETS: '["127.0.0.1/32","{{ docker_servicos_network_ip_address_start | default('172.18') }}.0.0/16","192.168.0.0/16"]'
      SVIX_JWT_SECRET: "{{ svix.jwt_secret }}"
      SVIX_MAIN_SECRET: "{{ svix.main_secret }}"
      SVIX_WHITELABEL_HEADERS: true
      SVIX_LOG_LEVEL: "info"
      SVIX_PORT: "{{ svix_port }}"
    networks:
      - servicos
```

- [ ] **Step 1: Criar `roles/svix/tasks/main.yml`**

```yaml
- name: Garantia do diretório de volume do Redis Svix
  ansible.builtin.include_role:
    name: utils
    tasks_from: services/ensure_volume_folder
  vars:
    volume_path: "{{ dados_docker_dir_path }}/svix_redis"

- name: Criação de banco de dados para SVIX
  ansible.builtin.include_role:
    name: utils
    tasks_from: create_db/postgres
  vars:
    db_name: "{{ svix_db_name }}"

- name: Configuração e transferência do compose do Svix
  ansible.builtin.template:
    dest: "{{ compose_dir_path }}/svix-compose.yml"
    src: composes/svix-compose.yml.j2
    owner: "{{ linux_korp.user }}"
    group: root
    mode: '0644'

- name: Criação e inicialização do Svix
  community.docker.docker_compose_v2:
    project_src: "{{ compose_dir_path }}/"
    env_files: ["{{ docker_env_file_path }}"]
    files:
      - svix-compose.yml
```
